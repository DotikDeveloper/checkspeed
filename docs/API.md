# API CheckSpeed

Все маршруты расположены под префиксом `/api`. Ответы не кэшируются (`Cache-Control: no-store`).

Rate limiting применяется через Next.js proxy (`src/proxy.ts`): **500 запросов в минуту** на IP. При превышении возвращается **429** с заголовками `Retry-After`, `X-RateLimit-Limit`, `X-RateLimit-Remaining`, `X-RateLimit-Reset`.

---

## `GET /api/download`

Генерирует бинарный payload заданного размера для измерения скорости **получения** (download).

### Параметры запроса

| Параметр | Тип   | Описание                                      |
|----------|-------|-----------------------------------------------|
| `size`   | число | Размер в мегабайтах (MiB). По умолчанию: `1` |

Допустимый диапазон: **0,5–10** МБ. Значения вне диапазона приводятся к границам.

### Ответ

- **200** — тело `application/octet-stream`, случайные байты (`randomFillSync`), без сжатия содержимого.
- **429** — превышен rate limit (JSON с полем `retryAfter`).

### Пример

```http
GET /api/download?size=2 HTTP/1.1
```

---

## `POST /api/upload`

Принимает тело запроса и возвращает фактический размер в байтах. Используется для измерения скорости **отдачи** (upload).

### Тело запроса

Произвольные бинарные данные. Максимум: **10 МБ** (10 × 1024² байт).

### Ответ

| Код | Тело |
|-----|------|
| **200** | `{ "size": <число байт> }` |
| **400** | `{ "error": "..." }` — неверный `Content-Length` или несовпадение с фактическим размером |
| **413** | `{ "error": "Payload exceeds allowed limit" }` |
| **429** | rate limit |
| **500** | внутренняя ошибка |

### Пример

```http
POST /api/upload HTTP/1.1
Content-Type: application/octet-stream
Content-Length: 2097152

<бинарные данные 2 МБ>
```

---

## `HEAD /api/ping`

Минимальный ответ для измерения задержки (RTT). Тело пустое.

### Ответ

- **204 No Content** — успех.
- **429** — rate limit.

### `GET /api/ping`

Возвращает JSON `{ "ok": true }` (вспомогательный endpoint; клиент использует **HEAD**).

---

## Заголовки rate limit (успешные ответы API)

| Заголовок | Описание |
|-----------|----------|
| `X-RateLimit-Limit` | Лимит запросов в окне |
| `X-RateLimit-Remaining` | Оставшиеся запросы |
| `X-RateLimit-Reset` | Unix timestamp (сек) сброса окна |
